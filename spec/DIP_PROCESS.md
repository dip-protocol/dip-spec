# DIP Improvement Proposal Process

The Deterministic Integrity Protocol (DIP) evolves through the DIP Improvement Proposal (DIP) process.

A DIP is a design document describing a change to the protocol.

## DIP Types

Standards Track  
Changes to protocol behavior or artifact structure.

Informational  
Documentation or explanatory improvements.

Process  
Changes to governance or workflow.

## DIP Lifecycle

Draft  
Proposed for discussion.

Review  
Community feedback and technical validation.

Accepted  
Approved for inclusion in the protocol.

Final  
Implemented in reference implementations.

## Proposal Format

Each proposal must include:

Title  
Author  
Status  
Abstract  
Motivation  
Specification  
Security Considerations
